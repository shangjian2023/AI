# ADR-0034: 提高对数似然展示最低证据门槛

- **状态**: Accepted
- **日期**: 2026-07-16
- **决策者**: 项目组
- **相关**: ADR-0033

## 背景

ADR-0033 最初把平均 token 对数似然差 `0.5` 作为宽松最低证据门槛。它对应候选序列
每个 token 的平均似然相对内部对照约提高 `exp(0.5) = 1.65` 倍，目的是只排除非常弱的
优化信号，再由候选族支持度完成主要过滤。该数值不是从 clean 分布统计拟合出的阈值。

现有 2 个后门与 5 个 clean 开发模型的最大对数似然差全部超过 `0.5`，因此该门槛在当前
集合上没有筛选作用。限制到族支持至少为 5 的同一候选后，两个后门模型分别保留
`2.9551、1.6990` 与 `2.2556、0.5865` 两组候选，5 个 clean 均没有达到族支持门槛的
候选。clean 的全局最大对数似然差仍可达到 `3.3111`，说明对数似然不能单独裁决。

## 决策

1. 新展示 profile `gpt2-loglikelihood-family-dev-v2` 将优化期最大平均 token 对数似然差
   的最低证据门槛提高为 `2.0`。
2. 最终判定仍要求同一候选同时满足对数似然差 `>= 2.0` 和共享 8-token 后缀候选族支持
   `>= 5`；禁止分别取两个候选的最大值后拼接结论。
3. `2.0` 对应每 token 平均似然倍率约 `exp(2.0) = 7.39`。它会排除 `1.6990` 与
   `0.5865` 的较弱同族候选，同时两个后门开发模型仍各保留一个决定性候选。
4. Competition Core 原始论文概率判据继续固定为 `0.25`，本次修改只影响竞赛展示层。
5. 该数值仍是小样本开发门槛，不是正式 clean 分位数阈值。v2 冻结后必须用未参与本次
   选择的新模型盲测，不得根据盲测结果继续移动门槛。

## 后果

- 对数似然条件从几乎无筛选作用的宽松门槛变为强效应最低门槛。
- 对数似然-only 仍会在 clean 上产生阳性，报告必须始终同时展示族支持。
- 历史 v1 报告保留原始 `0.5` 元数据；新扫描、校准档案、体验资格和竞赛前端使用 v2。
