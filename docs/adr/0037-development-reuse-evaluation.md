# ADR-0037: 允许开发样本复用为竞赛评估

- **状态**: Accepted
- **日期**: 2026-07-18
- **决策者**: 项目组
- **相关**: ADR-0024、0026、0033、0034、0036

## 背景

Competition Core 当前 `gpt2-loglikelihood-family-dev-v2` 展示规则由 2 个后门和 5 个 clean
开发模型共同确定。此前路线图还要求冻结参数后另留至少 1 个后门和 1 个 clean 作为盲测，
并把该盲测设为继续扩展模型族之前的阻塞项。在单张 RTX 4060 和竞赛交付周期下，这会让
已有完整报告无法直接承担比赛展示、回归比较和跨模型覆盖诊断。

参与过阈值选择的数据在统计定义上不能重新变成 blind；但竞赛展示并不等同于 ADR-0024
的 `formal_blind` 聚合。可以放宽的是样本用途和执行顺序，而不是改写样本来源事实。

## 决策

1. 已参与阈值、候选规则或算法选择的完整模型报告，允许复用为
   `development_reuse` 评估。它可以计算混淆矩阵、Precision、Recall、F1、FPR，参与前端
   展示、工程回归和跨模型覆盖比较。
2. `development_reuse` 结果必须同时记录或在相邻说明中明确
   `calibration_overlap=true`，并称为开发集重用指标；不得称为 blind、独立泛化或正式统计
   校准结果。样本复用不增加独立样本数。
3. Competition Core 不再把新增 1 个后门 + 1 个 clean 盲测设为继续展示、重跑 OPT-125M
   或扩展条件类型的前置阻塞项。独立新种子降为推荐增强证据。
4. OPT-125M 单个 matched pair 在取得 Adapter 并按 ADR-0036 重跑后，可以直接作为
   `development_reuse` 覆盖结果展示；不强制先补齐到 GPT-2 的 2 后门 + 5 clean 规模。
5. 单配对或小样本开发重用不能生成 OPT 专属正式阈值。需要声称跨种子稳定、独立泛化或
   模型族校准时，仍应增加 clean/backdoor 种子并在看到结果前冻结规则。
6. ADR-0024 的 `formal_blind` 定义和聚合器拒绝 development truth 的代码契约保持不变；
   ADR-0026 的 20-clean `formal` 校准门槛也保持不变。本 ADR 只调整 Competition Core
   竞赛证据的可用范围和优先级。
7. 若后来运行真正 blind 样本，仍必须保证其未参与阈值选择，并在结果出现后保持参数冻结；
   不能把 `development_reuse` 与 blind 混成一个未分层的总体指标。

## 后果

- 现有 GPT-2 2 后门 + 5 clean 可以直接作为当前竞赛开发重用矩阵，不再强制补 1+1 才能
  继续。
- 组员 OPT-125M 1 后门 + 1 clean 在权重补齐并重跑后即可形成一个开发重用配对结果；
  增加 1 后门 + 4 clean 不再是比赛展示的硬门槛。
- 对外材料必须保留“小样本、阈值与评估样本重叠”的限制说明。这样降低运行成本，但不会
  获得 blind 评估带来的独立泛化证据。
